The majority part of requests should contain authorization header. It should be passed in these requests in the Authorization header in the bearer format: Bearer YOUR_TOKEN.

Currently, we have 2 kind of tokens:

  • Refresh Token
    This kind of token is used for generating access token and revoking current token endpoints only.
    It will expire in one case only: if you don't use it for a while.
  • Access Token
    This token is used for the majority requests in our API except endpoints which were described on previous step.
We use JWT format for our tokens.

Getting a Refresh Token

You have to go to the section Profile → Settings → Chatex API in the @chatex_bot.

Retrieve an Access Token

Important: Access token has an expiration in 1 hour.

curl --request POST 'https://api.chatex.com/v1/auth/access-token' \
  --header 'Authorization: Bearer YOUR_REFRESH_TOKEN'
See endpoint details.
